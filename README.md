# MikroORM Snapshot Lifecycle Repro

This repo reproduces a MikroORM snapshot diff issue:
- https://github.com/mikro-orm/mikro-orm/issues/7607

## Requirements

- Node 24+
- Docker

## Reproduce

Install dependencies:

```bash
npm install
```

Start with a clean Postgres instance:

```bash
docker compose down -v --remove-orphans
docker compose up -d --wait
```

At this point, the repo contains:

- an initial migration generated by `migration:create --initial`
- the snapshot as written by `migration:create`
- no database schema yet

Apply the migration:

```bash
npx mikro-orm migration:up
```

Now inspect the changes introduced by `migration:up`:

```bash
git diff
```

On my machine this rewrites `src/migrations/.snapshot-snapshot_repro.json` with changes such as:

- table order changes
- `varchar(26) collate "C"` becoming `varchar(26)`
- `timestamptz` becoming `timestamptz(6)`
- `int` becoming `int4`
- index / foreign key metadata being reshaped to match the introspected database schema

## Reset

To rerun from scratch:

```bash
docker compose down -v --remove-orphans
git restore src/migrations/.snapshot-snapshot_repro.json
docker compose up -d --wait
npx mikro-orm migration:up

# Alternatively
git restore src/migrations/.snapshot-snapshot_repro.json
npx mikro-orm migration:fresh
```
